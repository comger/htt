# HTT (Hydro-Triple-Trust) 核心设计与泛化定义

## 1. 项目愿景与定位
HTT（Hydro-Triple-Trust）是一个旨在**引导式构建轻量级本体模型**，并通过该本体模型进行**时空数据可信治理与自适应诊断**的 Skill 及交互式软件系统。
本项目主要解决在数字孪生、智慧水利及物联网等场景下，因环境干扰、设备老化、物理故障等引发的传感器“数据失真”和“虚假警报”痛点。通过对数据的可信治理，为后续的预报、预警、预演和决策提供高可信的底层数据支撑。

## 2. 核心设计架构

系统设计以物理本体为基座，以三重审计规则为核心，以时域累积和自适应反演为智能诊断手段，形成完整的闭环数据治理体系。

### 2.1 H-Ontology (轻量级物理本体) 建模
系统通过引导式的交互，构建目标场景的高精度轻量级物理本体（H-Ontology），完成对客观物理世界的数字化复刻。本体模型确立了数据审计的“刚性边界”和“物理骨骼”。
* **空间地貌/环境本体**：包含诸如流域集雨面积、河道长度、断面几何剖面、警戒水位等自然环境关键属性。
* **工程设施/设备本体**：包含诸如大坝设计坝高、汛限水位、闸门尺寸、下泄流量关系曲线（G-H-Q）、设备工程极限等物理参数。

### 2.2 HTT “三道安全关” 数据审计规则
依托构建的轻量级本体模型，系统实施渐进式的三重数据治理与过滤规则：
1. **T1 物理硬限界 (Physical Limit - 单点静态物性拦截)**：
   基于物理本体定义的刚性值域空间，直接拦截超出工程或自然物理极限的“灵异野值”（例如水位读数超过大坝最高点）。
2. **T2 时序数理筛 (Time Series Statistical Filter - 单站时序常模校验)**：
   结合历史全局极限、季节性同期极限，以及信号的平稳性特征（如 3-sigma 原则），诊断并过滤信号的高频噪声或数据冻结现象。
3. **T3 因果机理钟 (Causal Mechanism Clock - 多源时空机理审计)**：
   引入多源数据的时空机理交叉审计。依据客观物理规律（如降雨与水位起涨的时滞关系、理论最大起涨速率、基流消落规律等）进行逻辑对账。

### 2.3 基于时域累积的自适应智能诊断
系统引入了时域累积校验因子（$\Delta t$），结合上述三道关的触发情况，对报警进行事件定性与精准诊断：
* **环境偶发干扰**：表现为 T2 瞬时报警，且持续时间 < $\Delta t$。系统判定为偶发噪声，自动进行滤波平滑，消除虚假警报。
* **系统性设备故障**：表现为 T1 或 T2 持续报警，且持续时间 ≥ $\Delta t$。系统判定为传感器或硬件故障，自动生成故障码并派发维护工单。
* **业务异常与隐蔽故障**：表现为 T1/T2 校验通过，但 T3 持续报警。系统通过机理逻辑进一步区分是隐蔽硬件故障（如雨量计被落叶堵塞导致机理不符）还是真实的极端业务场景（如发生真实洪水险情）。

### 2.4 孪生图谱与机理参数自适应反演
系统建立了“实测-反向纠偏”的闭环机制。通过自动比对现场实测数据与基于本体模型预测数据之间的残差，动态反演并修正孪生图谱中的机理参数（如渠道等效糙率、渗压比等）。这一自适应机制确保了本体模型能够持续学习，永远真实贴合现场的物理状态。

## 3. 核心设计原则 (轻量级约束)
为确保系统能够在广泛的边缘场景落地，项目在技术选型与架构设计上严格遵循“轻量级”原则：

### 3.1 极简算力与云边协同 AI 架构
* **云端辅助构建**：在开发和构建本体模型阶段，允许灵活配置并调用云端大模型 API 来辅助知识抽取和交互式引导构建。
* **边端自主运行**：应用和生成的模型需具备在普通 PC 或仅有少量算力（如 1~3Tops）的边缘 Linux 系统上独立运行的能力。在运行态，系统使用本地轻量级 AI 模型（如 Gamma 4 等端侧小模型）进行推理、时空数据预处理和自适应诊断，确保低延迟与数据隐私。

### 3.2 轻量级图谱与内存管理
* **边缘图谱存储**：采用极轻量级的图谱数据存储方案，支撑本体模型在端侧的持久化。
* **低内存开销**：允许直接在边端设备上进行图谱的管理（节点关系更新、动态属性调整等），并且整体运行内存占用必须控制在极小范围内，不影响边缘设备上的其他核心进程。

## 4. 泛化应用场景与价值
尽管 HTT 的理论原型起源于防洪场景下的水动力学规律与数字孪生流域建设，但其“**本体建模定义边界 + 多重规则交叉审计 + 时域逻辑自适应诊断**”的核心架构具备极强的跨领域泛化能力：
* **工业物联网 (IIoT) 与智能制造**：用于关键产线设备的健康监测、异常数据清洗与预防性维护。
* **智慧城市与基础设施基建**：用于桥梁、隧道、地下管网的结构健康状态传感器数据治理。
* **能源与电力系统**：用于风电、光伏及储能电站的海量测点数据清洗、故障识别与效能自适应诊断。
* **通用物联网数据中间件**：作为独立的 Data Governance Skill，为各类需要极高数据可信度的 AI/大模型应用提供可靠的数据底座。
